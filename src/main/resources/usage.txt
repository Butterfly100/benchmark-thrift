Usage: ./BT.sh -p <protocol file> [options] <host>:<port>/<service>/<method>
Options:
   -p thriftfile  File pointed the protocol and the transport and thrift version
   -d datafile    File containing data to the rpc method
   -c concurrency Number of multiple requests to make at a time
                  If no -c nor -q is specified, default value is 1 concurrency
   -q throughput  Number of requests issued in 1 Second
                  If no -c nor -q is specified, default value is 1 concurrency
   -D duration    Load test duration, a value of 10 or 10s means 10 seconds, 2m for 2 minutes
                  If not specified, default value is 60 seconds
   -h             Display usage information (this message) and exit
   -v             Print version number and exit"
Examples:
    # Benchmark at 1 concurrency for 60 seconds:
    ./BT.sh -p protocol.conf -d data.txt 127.0.0.1:8090/service/method
    # Benchmark at 10 QPS for 5 minutes:
    ./BT.sh -p protocol.conf -d data.txt -q 10 -D 5m 127.0.0.1:8090/service/method
